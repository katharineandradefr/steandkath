# Decisões arquiteturais (ADRs)

Registro de decisões importantes do projeto — **Architecture Decision Records**.

## Quando criar um ADR

- Mudança de stack ou biblioteca principal
- Decisão de modelo de dados (ex.: 3 → 5 status de pendência)
- Escolha entre alternativas com tradeoffs significativos
- Mudança de fluxo de auth ou permissões

## Formato sugerido

Nome do arquivo: `YYYY-MM-DD-titulo-curto.md`

```markdown
# Título da decisão

## Status
Proposta | Aceita | Substituída

## Contexto
Por que precisamos decidir?

## Decisão
O que foi escolhido?

## Consequências
Prós, contras, o que fica mais difícil/fácil
```
